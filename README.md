# SMARTCS_Pole 프로젝트 개요

## 1. 프로젝트 목적
SMARTCS_Pole은 전주 계측 데이터를 기반으로 파단 위험을 예측하기 위한 AI 파이프라인이다.  
데이터 수집/정제, 학습 데이터 생성, 경량 분류 모델(Light), 위치/신뢰도 모델(Hard 1/2차), 최종 결합 모델(MLP)까지 한 저장소에서 관리한다.

## 2. 시스템 구성
- 데이터 준비: 원천 CSV를 정규화하고 학습 가능한 텐서/라벨로 변환
- Light 모델: 파단/정상 이진 분류 중심의 1차 판단
- Hard 1차 모델: ROI 축(x/y/z)별 박스 예측 성능 학습
- Hard 2차 모델: 1차 결과를 기반으로 confidence head 보강
- MLP 모델: Light/Hard 출력을 결합한 최종 위험도 추정
- 결과 패키징: 모델/임계치/메타정보를 포함한 배포 패키지 생성

## 3. 디렉터리 개요
- `config/`: DB/환경 설정 및 DB 핸들러 모듈
- `main/1. make_data set/`: 데이터 수집·정제·학습셋 생성 관련 스크립트
- `main/2. make_light_model/`: Light 모델 학습/평가 및 베스트 모델 관리
- `main/3. make_hard_model/`: Hard 1차/2차 학습/평가 및 베스트 모델 관리
- `main/4. make_mlp_model/`: MLP 학습 및 최종 파이프라인 모델 생성
- `log/`: 공통 실행 로그 및 스크립트별 로그 파일

## 4. 베스트 모델 관리 원칙
- Light/Hard 1차/Hard 2차는 각각 별도의 `best_*` 디렉터리로 최신 베스트 모델 별칭을 관리한다.
- 베스트 교체/유지/스킵 이벤트는 히스토리 파일과 상세 변경 로그에 누적 기록된다.
- 상세 변경 로그에는 이전 대비 성능 변화(delta), 선택 기준, 교체 사유가 함께 저장된다.

## 5. 로그 체계
- 공통 `logger.py`를 통해 콘솔/파일 로그 포맷을 통일한다.
- 프로젝트 로그 규약은 `main/LOGGING_CONVENTION.md`에 고정한다.
- 로그 레벨은 `INFO/WARN/ERROR` 3단계로 사용한다.
- 메시지 키워드는 `MODEL_SELECT`, `EVAL_STAGE` 등 표준 키워드 집합을 사용한다.
- 학습 스크립트별 로그 파일이 분리되어 추적 가능하도록 구성한다.
- 베스트 모델 관련 의사결정은 모델별 상세 로그 파일로 별도 보관한다.

## 6. 산출물 특성
- 각 모델 run 디렉터리는 체크포인트, 평가 리포트, 시각화 결과를 포함한다.
- 최종 배포 패키지는 추론에 필요한 모델 파일, 임계치, 메타정보를 함께 포함한다.
- 평가 산출물은 성능 추이 분석 및 재학습 판단 근거로 활용된다.
