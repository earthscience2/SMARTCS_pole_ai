16. test_ai.py 외주 전달 가이드
============================================================

1) 목적
------------------------------------------------------------
이 문서는 외주 업체가 `make_ai/16. test_ai.py`와 베스트 모델 패키지를
웹 페이지/서비스에 연동할 때 필요한 기술 정보를 제공합니다.


2) 전달 파일 목록
------------------------------------------------------------
[필수]
1. `make_ai/16. test_ai.py`
2. 베스트 패키지 1개 (`make_ai/15. make_save_model/<package_name>/`)
   - `package_manifest.json`
   - `light_model/saved_model/`
   - `hard_model_2nd/saved_model/conf_x/`
   - `hard_model_2nd/saved_model/conf_y/`
   - `hard_model_2nd/saved_model/conf_z/`
   - `mlp_model/model/mlp_pipeline.joblib`
   - `mlp_model/test_results/mlp_summary.json`
3. DB 연동 모듈 (`config/poledb.py`와 동일 인터페이스)
4. 패키지 설치 파일: `make_ai/16_test_ai_requirements.txt`

중요:
- `16. test_ai.py`는 전처리/추론 로직이 파일 내부에 포함되어 있습니다.
- 동적 import 없이 단독 실행 가능하도록 구성되어 있습니다.


3) 권장 실행 환경
------------------------------------------------------------
[OS]
- Ubuntu 22.04 LTS 또는 Windows 11 + WSL2

[Python]
- Python 3.10.x 권장

[GPU 권장]
- NVIDIA GPU (VRAM 8GB 이상)
- 설치된 TensorFlow 버전에 맞는 CUDA/cuDNN

[CPU 모드 최소]
- 8코어 이상, RAM 16GB 이상

[운영 권장]
- 12코어 이상, RAM 32GB 이상, NVMe SSD

[필수 Python 패키지]
- tensorflow
- numpy
- pandas
- scikit-learn
- joblib
- openpyxl
- tqdm


4) 전처리 과정 (16번 내부 로직)
------------------------------------------------------------
1. DB에서 전주 측정 데이터 조회 (OUT x/y/z)
2. raw CSV 저장 (전주/측정번호 단위)
3. x/y/z 병합 후 `*_OUT_processed.csv` 생성
   - 출력 컬럼: `height, degree, x_value, y_value, z_value, devicetype`
4. processed CSV -> 모델 입력 텐서 변환
   - 필수 컬럼: `height, degree, x_value, y_value, z_value`
   - degree remap (90~180)
   - x/y/z 파일 단위 min-max 정규화
   - `(H,19,3)` 생성 후 `(304,19,3)` 리사이즈
5. 포인트 수 필터링
   - 기본: `--min-points 200`, `--max-points 400`


5) 모델 추론 방식
------------------------------------------------------------
입력:
- `(N, 304, 19, 3)` float32

추론 순서:
1. Light 모델 -> `light_prob`
2. Hard2(conf_x/conf_y/conf_z) -> 축별 confidence
3. 축 가중합 -> `hard_score`
4. MLP 입력 피처 7개 구성
   - `light_prob, conf_x, conf_y, conf_z, hard_score, fused_score, abs_gap`
5. MLP 출력 -> `mlp_prob_break`
6. 임계치 적용 (`mlp_summary.json`)
   - `break_threshold` 이상: `break`
   - `suspect_threshold` 이상: `suspect`
   - 그 외: `normal`


6) 실행 예시
------------------------------------------------------------
[프로젝트 단위]
python "make_ai/16. test_ai.py" ^
  --server main ^
  --project "강원원주-202112" ^
  --saved-package "20260212_234508" ^
  --saved-base-dir "15. make_save_model" ^
  --output-dir "16. test_ai" ^
  --device auto

[전주 1개 단위]
python "make_ai/16. test_ai.py" ^
  --server main ^
  --pole-id "0621R481" ^
  --saved-package "20260212_234508" ^
  --device auto


7) 출력 결과
------------------------------------------------------------
출력 경로:
- `make_ai/16. test_ai/<run_name>/`

주요 파일:
1. `prediction_<server>_<target>_<timestamp>.xlsx`
   - `by_measure` 시트: 측정 단위 결과
   - `by_pole` 시트: 전주 단위 집계
2. `run_summary.json`
3. `fetch_status.csv`

실제값 컬럼:
- `actual_breakstate`, `actual_label`, `actual_label_num`

예측값 컬럼:
- `pred_label`, `pred_label_num`, `mlp_prob_break`


8) 웹 연동 권장
------------------------------------------------------------
권장 구조:
- 프론트엔드 -> 백엔드 API -> `16. test_ai.py` 실행 -> 결과 경로/상태 반환

권장 API:
1. `POST /api/predict/project`
2. `POST /api/predict/pole`
3. `GET /api/predict/result/{run_name}`

운영 권장:
- 장시간 작업은 비동기 큐로 실행
- 진행률 상태를 저장해 UI에 표시


9) 검증 체크리스트
------------------------------------------------------------
1. `package_manifest.json` 로드 성공
2. SavedModel 4개 로드 성공
3. `mlp_pipeline.joblib` 로드 성공
4. 샘플 1건 추론 성공 (`(1,304,19,3)`)
5. 엑셀 2개 시트 생성 확인 (`by_measure`, `by_pole`)
6. 실제값/예측값 컬럼 존재 확인
