# 데이터 머지 프로세스 분석 및 개선 방안

## 📊 원본 데이터 분석

### 데이터 구조
- **원본 파일 형식**: `{poleid}_{index}_{date}_OUT_{axis}_breakheight_{height}_breakdegree_{degree}.csv`
- **컬럼 구조**: `idx, groupname, poleid, devicetype, measno, ch1, ch2, ch3, ch4, ch5, ch6, ch7, ch8, axis`
- **데이터 특성**: 각 행은 하나의 높이에서 측정된 8개 센서(채널) 데이터

### 해상도 분석 (0621R481 예시)

| 측정 | 각도 범위 | 높이 범위 | 행 수 | 높이 해상도 | 높이 간격 |
|------|----------|----------|-------|------------|----------|
| OUT_1 | 0~90도 | 0.0~1.5m | 222행 | **6.76mm/행** | 0.00676m |
| OUT_2 | 90~180도 | 0.0~1.5m | 281행 | **5.34mm/행** | 0.00534m |
| OUT_3 | 90~180도 | 0.0~1.5m | 378행 | **3.97mm/행** | 0.00397m |
| OUT_4 | 180~270도 | 0.0~1.5m | 245행 | **6.12mm/행** | 0.00612m |
| OUT_5 | 270~360도 | 0.0~1.5m | 242행 | **6.20mm/행** | 0.00620m |

**평균 높이 해상도**: 약 **5.5mm/행** (0.0055m)

### 채널 데이터 특성
- **8개 채널 (ch1~ch8)**: 각 채널은 측정 각도 범위 내에서 특정 각도에 매핑
- **채널-각도 매핑**: 
  - 측정 범위 0~90도 → ch1=10도, ch2=20도, ..., ch8=80도
  - 측정 범위 90~180도 → ch1=100도, ch2=110도, ..., ch8=170도
- **데이터 범위**: 채널별로 -0.19 ~ 0.08 정도의 값 범위

## 🔍 현재 머지 방식 분석

### 현재 보간 설정
- **높이 간격**: 10cm (0.1m)
- **각도 간격**: 5도
- **결과**: 높이 16개 × 각도 73개 = 1,168행

### 현재 처리 방식
1. **높이 방향 보간**: 
   - 원본 데이터의 각 행을 높이로 매핑 (선형 가정)
   - `interp1d`로 높이 방향 선형 보간
   - **문제점**: 원본 해상도(5.5mm) → 목표 해상도(100mm)로 약 **18배 정보 손실**

2. **각도 방향 보간**:
   - 8개 채널을 각도에 매핑 (ch1=10도, ch2=20도, ...)
   - `interp1d`로 각도 방향 선형 보간
   - **문제점**: 8개 채널만으로 각도 보간 → 각도 해상도 제한

3. **채널 데이터 집계**:
   - 8개 채널을 각도 보간으로 하나의 값으로 변환
   - **문제점**: 채널별 정보 손실

## ⚠️ 현재 방식의 문제점

### 1. 높이 해상도 손실
- **원본**: 5.5mm 간격 (약 273개 높이 포인트)
- **머지**: 100mm 간격 (16개 높이 포인트)
- **손실률**: 약 **94%** (273개 → 16개)

### 2. 채널 정보 손실
- 원본에는 8개 채널의 개별 데이터가 있음
- 머지 후에는 하나의 값(x_value, y_value, z_value)만 남음
- 채널별 특성 정보 손실

### 3. 보간 방법의 한계
- **1D 보간 (높이) → 1D 보간 (각도)** 순차적 처리
- 실제로는 **2D 데이터 (높이 × 각도)**이므로 2D 보간이 더 적합할 수 있음

### 4. 각도 보간의 부정확성
- 8개 채널만으로 각도 보간 → 각도 해상도 제한
- 측정 범위가 겹치는 경우 (예: OUT_2와 OUT_3 모두 90~180도) 처리 방식 불명확

**목표**: 높이와 각도를 동시에 고려한 보간

```python
    (target_heights, target_degrees),  # 목표 그리드

```

**장점**:

- 더 정확한 보간 결과

**단점**:

- 처리 시간 증가

## 🎯 권장 개선 사항

### 우선순위 1: 높이 해상도 개선
- **10cm → 1cm 간격**으로 변경
- 원본 데이터의 높이 해상도(5.5mm)를 고려하면 1cm 간격이 적절
- 데이터 크기: 16개 → 151개 높이 포인트 (약 9.4배 증가)

### 우선순위 2: 2D 보간 방법 도입
- `scipy.interpolate.griddata` 또는 `scipy.interpolate.RectBivariateSpline` 사용
- 높이와 각도를 동시에 고려한 보간

### 우선순위 3: 보간 방법 개선
- **선형 보간 (linear)**: 현재 사용 중, 빠르지만 부드러움 부족
- **3차 보간 (cubic)**: 더 부드러운 결과, 하지만 경계 처리 주의 필요
- **스플라인 보간 (spline)**: 부드러운 곡선, 계산 비용 높음

### 우선순위 4: 채널 데이터 처리 개선
- 현재는 8개 채널을 각도 보간으로 하나의 값으로 변환
- 채널별 특성을 고려한 집계 방법 고려 (평균, 중앙값, 가중 평균 등)
